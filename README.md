# AI TradeBot - 事件驱动自动化交易系统

> "Begin with the End in Mind" - 从买入那一刻起，AI已预设明确的逻辑终点

## 🎯 项目愿景

本系统是一个基于**事件驱动**的智能化交易系统，核心哲学是"以终为始"。系统不仅负责发现市场机会，更关键的是在每一次交易决策的初始，就由 AI 推演并预设清晰的退出策略：

- **止盈目标** (Take Profit) - 基于市场情绪与逻辑强度的收益预期
- **止损边界** (Stop Loss) - 逻辑证伪时的风险控制线
- **失效时间** (Expiration) - 事件逻辑的时间窗口

## 🧩 核心模块说明

### 1. Perception Layer (感知层)

| 模块 | 职责 | 技术 |
|------|------|------|
| **OpenClaw** | 公告爬虫、网站数据提取 | Playwright 无头浏览器 |
| **Tavily** | 实时新闻搜索、舆情监控 | Tavily Search API |
| **Tushare** | 历史行情、基本面数据 | Tushare Pro API |
| **AkShare** | 实时行情数据 | AkShare |

### 2. Decision Layer (决策层)

#### AI Matrix 多模型协同

| AI 模型 | 核心能力 | 应用场景 |
|---------|----------|----------|
| **Kimi** | 长文本理解、摘要 | 公告清洗、研报提取 |
| **GLM-4** | 逻辑推演、博弈分析 | 退出策略、情景分析 |
| **MiniMax** | 结构化指令生成 | 标准化决策输出 |
| **Tavily AI** | 实时信息聚合 | 事件背景搜索 |

#### Decision Engine (决策引擎)

```
决策引擎负责协调 AI Matrix 的多模型协同，工作流程：

1. 信号生成 (Signal Generation)
   - 基于感知层数据生成交易信号

2. 逻辑推演 (Reasoning)
   - 多空逻辑分析
   - 风险评估
   - 市场情景推演

3. 退出规划 (Exit Planning)  ← "以终为始" 核心模块
   - 计算止盈目标 (基于逻辑强度)
   - 设定止损边界 (基于逻辑证伪点)
   - 确定失效时间 (基于事件时间窗口)

4. 决策打包 (Bundle)
   - 生成完整 Decision Bundle
   - 包含完整推理链路
```

### 3. Risk Control Layer (风控层)

**硬编码风控锁 (Safety Locks)**

- **仓位锁**: 单一标的最大仓位限制
- **资金锁**: 日/周/月最大亏损熔断
- **频率锁**: 单日最大交易次数
- **时间锁**: 盘中/盘外交易时段控制
- **标的锁**: 黑名单/白名单机制

### 4. Execution Layer (执行层)

| 组件 | 职责 |
|------|------|
| **QMT Gateway** | 与券商柜台系统通信 (xtquant) |
| **Order Router** | 订单路由、拆单、限价/市价处理 |
| **Position Manager** | 持仓实时跟踪、盈亏计算 |

### 5. Storage Layer (存储层)

**数据库表设计**

| 表名 | 用途 |
|------|------|
| `events` | 原始事件记录 |
| `decisions` | AI 决策记录（含完整推理链） |
| `positions` | 当前持仓 |
| `trade_log` | 完整交易历史 |
| `ai_reasoning` | AI 原始推理过程（可追溯） |

### 6. UI Layer (展示层)

**Streamlit 监控面板功能**

- 实时行情监控
- AI 决策溯源展示
- 持仓盈亏分析
- 交易日志查询
- 回复盘览

## 🚀 快速开始

### 前置要求

- Python 3.10+
- QMT 交易终端
- 各 AI 服务的 API Key

### 安装

```bash
# 1. 克隆项目
git clone https://github.com/your-repo/aitradebot.git
cd aitradebot

# 2. 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate  # Windows

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置环境变量
cp .env.example .env
# 编辑 .env 填入你的 API Keys

# 5. 初始化数据库
python scripts/init_db.py

# 6. 启动服务
# 终端 1: 启动 FastAPI 后端
uvicorn core.main:app --reload

# 终端 2: 启动感知层（爬虫/数据采集）
python perception/main.py

# 终端 3: 启动 UI 面板
streamlit run ui/dashboard.py
```

## 🔐 安全与风控

### 硬编码风控锁

**位置**: `core/security.py`

```python
# 风控参数（硬编码，不可通过配置修改）
MAX_SINGLE_POSITION_RATIO = 0.10  # 单只股票最大仓位 10%
MAX_DAILY_LOSS_RATIO = 0.03       # 单日最大亏损 3%
MAX_WEEKLY_LOSS_RATIO = 0.05      # 单周最大亏损 5%
MAX_TRADES_PER_DAY = 10           # 单日最大交易次数
MIN_ACCOUNT_BALANCE = 50000       # 最低账户余额要求

# 熔断触发后暂停交易时长
CIRCUIT_BREAK_COOLDOWN_HOURS = 24
```

### API Key 管理

- 所有 API Key 通过环境变量配置
- 敏感信息不提交到代码仓库
- `.env` 文件加入 `.gitignore`

## 📊 透明性与可追溯性

### AI 推理日志

所有 AI 模型的原始推理过程都会存储到 `ai_reasoning` 表：

```python
{
    "decision_id": "DEC_20250111_001",
    "ai_model": "kimi",
    "timestamp": "2025-01-11 10:30:00",
    "input_text": "原始输入...",
    "output_text": "AI输出...",
    "prompt_used": "使用的提示词...",
    "reasoning_process": "完整推理链...",
    "metadata": {...}
}
```

### 决策溯源

在 UI 面板中，每一笔交易都可以追溯到：
- 触发事件
- AI 决策过程
- 风控检查结果
- 执行详情
- 退出原因
